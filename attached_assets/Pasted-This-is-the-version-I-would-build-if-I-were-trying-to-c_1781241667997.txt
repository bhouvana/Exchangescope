# REPLIT PROJECT PROMPT

## Project Name

**ExchangeScope ŌĆō Interactive Stock Exchange & Matching Engine Visualizer**

---

## Project Vision

Build a professional web application that demonstrates how modern stock exchanges work internally.

The platform should not be a stock trading platform.

The platform should not be a Bloomberg clone.

The platform should be a financial systems visualization laboratory.

Users should be able to see exactly how orders move through an exchange in the same way a CPU simulator visualizes instructions moving through a processor pipeline.

The application should combine:

* Real market data using yfinance
* Custom C++ matching engine
* Realtime order book visualization
* Exchange pipeline animation
* AI-powered explanations
* Market simulation scenarios

The final product should feel like:

```text
CPU Simulator
+
TradingView
+
NASDAQ Matching Engine Visualization
```

---

# PRIMARY OBJECTIVE

The main feature of the application is not stock prices.

The main feature is visualizing what happens inside an exchange after an order is submitted.

Do NOT implement matching logic in JavaScript.

The matching engine must be written entirely in modern C++.


5. Pipeline visualization

Do not generate placeholder code. Produce working, production-quality implementations for each phase before moving to the next.
